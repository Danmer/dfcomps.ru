Правила Speedrun DeFrag Cup

1. Действуют общие правила проведения турниров (разделы 1-4 правил dfcomps)

2. Турнир проходит в формате Single Elimination.

3. Определение победителя

3.1 Победителем матча становится игрок, быстрее всего прошедший выбранный набор карт. Итоговое время считается по сумме времени прохождения каждой отдельной карты.

3.2 Время прохождения карты считается от первого момента срабатывания стартового триггера до срабатывания финишного триггера. При этом, если время от появления игрока на карте до срабатывания первого стартового триггера больше чем 3 секунды, избыток добавляется к общему времени. Например, если игрок потратил 10 секунд на преджамп, к его итоговому времени на карте добавятся 10 - 3 = 7 секунд. Если игрок потратит на преджамп меньше трех секунд, то избыточного времени не будет.

4. Процесс выбора карт

4.1 Полный список карт объявляется заранее.

4.2 Процесс выбора карт осуществляется по ссылке, доступной в начале матча.

4.3 Игроки по очереди банят карты, пока не останется набор карт в количестве определенном в туре.

4.4 Список карт во время бана соответствует тому порядку, в котором карты нужно будет пройти. Список карт для каждого матча определяется случайной перетасовкой.

5. Проведение матча

5.1 Игроки подключаются к разным выделенным серверам, ссылку на сервер получают через сайт dfcomps.

5.2 Игрокам нужно поставить параметр name такой же, под которым они зарегистрированы на турнире и написать !ready в консоль для обозначения готовности.

5.3 Смена карт и подведение результатов будет происходить автоматически, игрокам не нужно делать callvote.

5.4 Игрок может сдаться до конца матча, написав в консоли !giveup .

6. В случае возникновения спорных ситуаций, окончательное решение принимают организаторы.

Speedrun DeFrag Cup rules

1. Common tournament ruleset applies (sections 1–4 of the dfcomps rules)

2. The tournament is held in Single Elimination format.

3. Determining winner

3.1 The winner of the match is the player with the fastest time through all the selected maps. The final time is the sum of times of each individual map.

3.2 The map time is counted from the first moment of the start trigger to the finish trigger. If time from entering the game to start trigger intersection exceeds 3 seconds, additional time is added to the final results. For example, if a player spends 10 seconds prejumping, 10 - 3 = 7 seconds are added to his time on the map. If the player spends less than three seconds on the prejump, there are no additional time.

4. Map selection process

4.1 The full list consists is published before tournament.

4.2 Map banning takes place by link provided before the match.

4.3 Players take turns banning maps until there are required amount of maps left, each player has 2 bans.

4.4 The list of maps during the ban phase corresponds to the order in which the maps are played. The list of maps for each match is determined by random shuffling.

5. Match process

5.1 Players connect to different dedicated servers, the server link is obtained through the dfcomps website.

5.2 Players need to set the name parameter equal to name under which they are registered for the tournament and write !ready in the console.

5.3 Changing maps and summarizing results are automatic, players do not need to make a callvote.

5.4 A player can give up before the end of the match by writing !giveup in the console.

6. In case of disputes, the final decision is made by the organizers.
